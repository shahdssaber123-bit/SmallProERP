# 🧩 SmallProERP

SmallProERP is a simple ERP (Enterprise Resource Planning) system designed to simulate real-world business operations in a simplified and educational way.

The system connects multiple business roles such as Admins, Suppliers, Factories, Drivers, and Hub Managers to manage resources, orders, and logistics in one organized platform.

---

## 📌 Project Idea

In real companies, multiple parties work together in a connected workflow:

- Admin manages the system
- Suppliers provide materials or products
- Factories handle production
- Hubs store and distribute items
- Drivers handle delivery between all parties

SmallProERP simulates this workflow to show how an ERP system coordinates between different entities in a business environment.

---

## 🧑‍💼 System Roles

### 👑 Admin
The Admin is the main controller of the system. Responsibilities include:
- Managing all users in the system
- Monitoring all operations
- Controlling permissions and access

---

### 🏭 Factory
The Factory is responsible for:
- Receiving production requests
- Manufacturing or preparing products
- Sending products to hubs or suppliers

---

### 📦 Supplier
The Supplier is responsible for:
- Providing raw materials or products
- Updating stock availability
- Handling incoming system requests

---

### 🚚 Driver
The Driver handles logistics and transportation:
- Delivering products between system entities
- Updating delivery status (Pending / In Progress / Delivered)

---

### 🏢 Hub Manager
The Hub Manager is responsible for:
- Managing warehouse inventory
- Receiving and storing products
- Distributing items based on requests

---

## ⚙️ How the System Works

The workflow of the system follows these steps:

1. The Admin manages and monitors the system
2. Suppliers or Factories prepare the required products
3. Products are sent to Hubs for storage
4. Drivers transport products between entities
5. Hub Managers control storage and distribution
