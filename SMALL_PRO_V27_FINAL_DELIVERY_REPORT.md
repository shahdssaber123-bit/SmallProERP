# Small Pro ERP V27 Final Delivery

Visible changes included in this package:

- V26/V27 Real Backend Fix badge appears in every page header so you can immediately see this is not the old UI.
- Removed fake calls to `/api/users/all`, `/api/users`, `/api/all` and single `/api/dashboard` from the frontend source.
- Dashboard reads from the real dashboard sub-endpoints such as sales overview, inventory overview, CRM pipeline, purchase overview, recent invoices, and low stock products.
- AI Generate modal now includes `InsightType`, `FromDate`, and `ToDate`, and posts `{ InsightType, FromDate, ToDate }` to `/api/insights/generate`.
- OCR upload uses multipart field name `image` for `/api/ocr/upload`.
- CRM Kanban uses backend customers and `PATCH /api/customers/{id}/status` with enum number.
- Quotation actions use the correct backend endpoints for status, add item, remove item, and convert.
- Supplier details use `/api/suppliers/{id}/details`.
- Purchase Order document uses `/api/purchase-orders/{id}/document`.
- Dropdowns/selects and modal layers were raised to high z-index to stop lists from opening behind popups.
- Build verified with `npm run build`.

Important backend note:
The uploaded backend source/Swagger does not expose a real GET users list endpoint. The Users page therefore does not call a fake users list and instead provides available Auth actions: register user, reset password, and current authenticated user.
